# Playwright Project Folder Structure

## Standard Base Structure

```
my-playwright-project/
├── playwright.config.ts          # Main Playwright configuration
├── global-setup.ts               # Runs once before all tests (auth, DB seed)
├── global-teardown.ts            # Runs once after all tests (cleanup)
├── package.json
├── tsconfig.json
│
├── tests/                        # All spec files (.spec.ts)
│   ├── auth/
│   │   ├── login.spec.ts
│   │   └── logout.spec.ts
│   ├── dashboard/
│   │   └── dashboard.spec.ts
│   ├── settings/
│   │   └── settings.spec.ts
│   └── api/                      # API REST tests (no browser)
│       ├── auth.spec.ts
│       └── users.spec.ts
│
├── pages/                        # Page Object Model classes
│   ├── LoginPage.ts
│   ├── DashboardPage.ts
│   └── SettingsPage.ts
│
├── components/                   # Reusable component Page Objects
│   ├── NavBar.ts
│   ├── Modal.ts
│   └── DataTable.ts
│
├── mocks/                        # HAR files and static mocks for api-mocking
│   └── api.har
│
├── fixtures/                     # Custom Playwright fixtures
│   ├── index.ts                  # Re-exports all fixtures (single import point)
│   ├── auth.fixture.ts           # Authenticated page/user fixtures
│   └── database.fixture.ts       # DB seed/cleanup fixtures
│
├── utils/                        # Pure helper functions and builders
│   ├── test-data-builder.ts      # Factory functions to build test data
│   ├── date-helpers.ts
│   ├── api-helpers.ts            # Direct API calls for test setup
│   └── schema-validator.ts       # JSON schema validation helpers
│
├── test-data/                    # Static test assets
│   ├── users.json                # Static user data
│   ├── products.csv
│   ├── images/
│   │   └── test-avatar.png
│   └── auth/
│       └── .gitkeep              # auth.json goes here (gitignored)
│
├── .github/
│   └── workflows/
│       └── playwright.yml        # CI pipeline
│
└── playwright-report/            # Generated HTML report (gitignored)
```

---

## Folder Purposes

| Folder | Purpose |
|--------|---------|
| `tests/` | All `.spec.ts` files. Organized by feature or page. No logic—only test scenarios. |
| `tests/api/` | API REST specs using `request` context of Playwright — no browser required. |
| `pages/` | Page Object classes. One class per full page of the app. |
| `components/` | Reusable Page Objects for UI components that appear on multiple pages (nav, modals, tables). |
| `mocks/` | HAR files recorded and static responses for `page.routeFromHAR()` and API mocking. |
| `fixtures/` | Custom Playwright fixtures extending `test`. Groups setup/teardown logic reusable across tests. |
| `utils/` | Pure TypeScript helpers. Data builders, formatters, direct API utilities. Not Playwright-specific. |
| `test-data/` | Static files referenced in tests: JSON datasets, images, CSVs, downloaded artifacts. |
| `playwright-report/` | Auto-generated by Playwright. Always gitignore this folder. |

---

## Root Files

| File | Purpose |
|------|---------|
| `playwright.config.ts` | Browser projects, base URL, retries, reporters, timeouts |
| `global-setup.ts` | One-time setup before all tests (e.g., authenticate and save `auth.json`) |
| `global-teardown.ts` | One-time cleanup after all tests |
| `tsconfig.json` | TypeScript config, should include path aliases for clean imports |

## TypeScript Path Aliases (recommended)

Configure `tsconfig.json` for cleaner imports:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@pages/*": ["pages/*"],
      "@components/*": ["components/*"],
      "@fixtures/*": ["fixtures/*"],
      "@utils/*": ["utils/*"],
      "@test-data/*": ["test-data/*"]
    }
  }
}
```

Then in tests:

```typescript
// Instead of: import { LoginPage } from '../../pages/LoginPage'
import { LoginPage } from '@pages/LoginPage';
```

## Quick Reference

| What you're creating | Where it goes |
|---------------------|---------------|
| A new test scenario | `tests/<feature>/<name>.spec.ts` |
| An API REST test (no browser) | `tests/api/<endpoint>.spec.ts` |
| A new page interaction class | `pages/<Name>Page.ts` |
| A nav bar / modal / table abstraction | `components/<Name>.ts` |
| A custom test fixture | `fixtures/<name>.fixture.ts` |
| A data factory / helper function | `utils/<name>-helpers.ts` |
| A JSON schema validator | `utils/schema-validator.ts` |
| A JSON/CSV/image used in tests | `test-data/<category>/` |
| Auth state file | `test-data/auth/auth.json` (gitignored) |
| A mock HAR / response static | `mocks/<name>.har` |
